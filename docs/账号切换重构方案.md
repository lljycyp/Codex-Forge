# Codex 多账号切换器重构方案

## 1. 目标

当前项目从“多开启动器”收敛为“多账号切换器”。

新的核心模型：

```text
默认安装的 Codex 程序
多份账号资料
一个当前激活账号
```

切换账号时，只把目标账号保存的 `auth.json` 和 `config.toml` 写入当前用户的 `.codex` 目录，然后按需执行 `codex app`。

## 2. 明确不做

- 不复制完整 Codex 程序。
- 不维护每个账号的 `CodexPortableApp` 程序副本。
- 不做会话同步。
- 不做记忆同步。
- 不做多个 Codex 客户端并发运行。
- 不做手动配置 Codex 程序路径。
- 不做程序副本版本比较。

## 3. 资料目录

账号根目录默认仍使用：

```text
%USERPROFILE%\Documents\CodexProfiles
```

目录结构：

```text
CodexProfiles
├─ 工作账号
│  ├─ auth.json
│  └─ config.toml
├─ 备用账号
│  ├─ auth.json
│  └─ config.toml
```

当前阶段继续沿用配置文件中的 `profiles` 列表，避免一次性迁移到新账号库导致历史数据不可读。

## 4. 活动配置

活动配置固定为：

```text
%USERPROFILE%\.codex\auth.json
%USERPROFILE%\.codex\config.toml
```

账号切换只操作这两个文件。

## 5. 账号操作

### 5.1 新增账号

新增账号时，读取当前活动配置：

```text
%USERPROFILE%\.codex\auth.json
%USERPROFILE%\.codex\config.toml
```

复制到账号目录：

```text
CodexProfiles\<账号名>\auth.json
CodexProfiles\<账号名>\config.toml
```

如果当前活动认证文件不存在，则拒绝新增，并提示先用默认 Codex 完成登录。

### 5.2 切换账号

切换流程：

```text
1. 校验账号目录存在。
2. 校验账号 auth.json 存在。
3. 检查是否已有 Codex 进程运行。
4. 若有运行进程，则提示用户确认关闭并切换。
5. 用户确认后关闭当前运行的 Codex 进程。
6. 备份当前活动 auth.json 和 config.toml。
7. 将目标账号资料写入活动 .codex 目录。
8. 记录当前激活账号。
9. 执行 codex app。
```

### 5.3 删除账号

删除账号时只删除账号资料目录和配置列表记录，不处理会话、记忆或程序副本。

### 5.4 改名账号

改名时移动账号资料目录，并更新配置列表。

## 6. 后端模块边界

保留：

- `config_store.py`：配置读写、备份恢复。
- `profile_service.py`：账号资料复制、应用、状态检查。
- `commands.py`：桥接命令。
- `codex_source.py`：保留进程读取和 Codex 启动来源识别能力。

废弃：

- `sync_service.py` 已从当前实现移除。
- 程序副本准备、更新和版本比较。
- 手动设置 Codex 程序路径。

## 7. 前端调整

页面语义调整为：

- “启动”改为“切换并启动”。
- “运行中”表示当前存在 Codex 进程，不再区分账号副本。
- “程序副本”改为“账号资料”。
- 设置页删除 Codex 路径和同步开关。
- 不保留独立诊断页，账号资料状态直接在账号列表展示。

## 8. 排障信息

后端可以保留轻量排障数据，供启动失败、调试命令或后续复制排障信息时使用。排障数据只检查：

- 配置文件是否存在。
- 配置备份是否存在。
- 账号根目录是否可写。
- 当前活动 `auth.json` 是否存在。
- 当前活动 `config.toml` 是否存在。
- 默认 `codex app` 命令是否可用。
- 是否存在运行中的 Codex 进程。
- 每个账号资料是否完整。

## 9. 迁移策略

旧目录不自动删除。

如果旧账号目录下存在：

```text
CodexHome\auth.json
CodexHome\config.toml
```

或旧版独立环境中存在可识别的认证文件，后续可以做一次性迁移工具提取到账号目录根部。

当前第一阶段先实现新的新增、切换、启动和账号列表状态主链路。
