# CodexMultiLauncher 开发与打包说明

本文档说明当前 uv 工程结构下的运行、依赖、虚拟环境和打包流程。

## 项目结构

```text
CodexMultiLauncher
├─ src
│  └─ codex_multi_launcher
│     ├─ __init__.py
│     ├─ app.py
│     └─ main.py
├─ scripts
│  ├─ build_pyd.ps1
│  ├─ clean.ps1
│  └─ run.ps1
├─ assets
│  └─ app.ico
├─ docs
│  ├─ 使用说明.md
│  └─ 开发打包说明.md
├─ pyproject.toml
├─ uv.lock
└─ README.md
```

核心源码位于：

```text
src\codex_multi_launcher\app.py
```

## 环境要求

- Windows 10 / Windows 11
- Python 3.11 或更高版本
- uv
- Codex 桌面端 `Codex.exe`

检查 uv：

```powershell
uv --version
```

检查 Python：

```powershell
uv run python --version
```

## 虚拟环境

本项目使用 uv 管理虚拟环境和依赖。

初始化普通运行环境：

```powershell
uv sync
```

初始化开发和打包环境：

```powershell
uv sync --dev
```

uv 会在项目根目录创建 `.venv`。该目录不需要提交。

## 依赖说明

启动器运行时只依赖 Python 标准库，不需要第三方运行依赖。

开发依赖放在 `pyproject.toml` 的 `dev` 依赖组中：

```toml
[dependency-groups]
dev = [
    "pyinstaller>=6.14.0",
]
```

更新锁文件：

```powershell
uv lock
```

## 运行源码

推荐使用项目入口命令：

```powershell
uv run codex-multi-launcher
```

也可以使用脚本：

```powershell
.\scripts\run.ps1
```

或者直接运行模块文件：

```powershell
uv run python .\src\codex_multi_launcher\app.py
```

## 语法检查

```powershell
uv run python -m py_compile .\src\codex_multi_launcher\app.py
```

如果没有输出，表示语法检查通过。

## 打包 exe

推荐使用脚本：

```powershell
.\scripts\build_pyd.ps1
```

脚本会执行：

1. `uv sync --dev`
2. `uv run setup_pyd.py build_ext --inplace`
3. `uv run pyinstaller --onefile --windowed --icon <图标路径> --add-data <图标资源> --name CodexMultiLauncher src\codex_multi_launcher\main.py --noconfirm --specpath build`
4. `uv run clear_pyd.py`
5. 生成 `dist\CodexMultiLauncher.exe`

手动调试打包命令：

```powershell
uv sync --dev

uv run setup_pyd.py build_ext --inplace

uv run pyinstaller `
  --onefile `
  --windowed `
  --icon "$PWD\assets\app.ico" `
  --add-data "$PWD\assets\app.ico;assets" `
  --name CodexMultiLauncher `
  src\codex_multi_launcher\main.py `
  --noconfirm `
  --specpath build

uv run clear_pyd.py
```

## 可选：管理员权限

如果需要打包后的程序启动时请求管理员权限，可以在 PyInstaller 参数里加入 `--uac-admin`：

```powershell
uv run pyinstaller `
  --onefile `
  --windowed `
  --uac-admin `
  --name CodexMultiLauncher `
  src\codex_multi_launcher\main.py
```

普通用户目录下运行通常不需要管理员权限；如果要访问或复制受保护目录，管理员权限可能更稳。

## 图标资源

项目图标位于：

```text
assets\app.ico
```

打包时必须同时传入 `--icon` 和 `--add-data`：

```powershell
uv run pyinstaller `
  --onefile `
  --windowed `
  --icon "$PWD\assets\app.ico" `
  --add-data "$PWD\assets\app.ico;assets" `
  --name CodexMultiLauncher `
  src\codex_multi_launcher\main.py
```

`--icon` 用于写入可执行文件自身图标；`--add-data` 用于让窗口运行时也能读取同一个图标资源。

## 清理构建产物

推荐使用脚本：

```powershell
.\scripts\clean.ps1
```

脚本会清理：

- `build`
- `dist`
- `*.spec`
- `__pycache__`

## 运行逻辑概览

启动器主要流程：

1. 读取配置文件 `config.json`。
2. 如果主配置损坏，尝试从最近两份备份恢复。
3. 自动查找微软商店版 `Codex.exe`。
4. 用户新增、导入、改名或删除账号。
5. 启动账号时创建独立目录。
6. 为该账号设置独立环境变量。
7. 新增或导入账号时为该账号准备独立 `CodexPortableApp` 程序副本。
8. 启动账号前检查微软商店版是否更新，必要时更新账号程序副本。
9. 使用独立 `--user-data-dir` 启动 Codex。
10. 通过进程命令行判断账号是否正在运行。
11. 通过诊断报告检查配置、路径、权限和账号目录状态。

## 账号隔离目录结构

单个账号目录大致如下：

```text
CodexProfiles
└─ 工作号
   ├─ AppData
   │  ├─ Local
   │  └─ Roaming
   │     └─ Codex
   │        └─ web
   │           └─ Codex
   ├─ CodexHome
   └─ CodexPortableApp
      └─ Codex.exe
```

## 常见问题

### 1. `uv` 找不到

先安装 uv，然后重新打开 PowerShell。

安装方式可以参考 uv 官方文档；如果本机已安装，`uv --version` 应能输出版本号。

### 2. 打包时提示找不到 PyInstaller

执行：

```powershell
uv sync --dev
```

然后重新运行：

```powershell
.\scripts\build_pyd.ps1
```

### 3. 打包后 exe 打不开或闪退

可以先用源码运行确认问题：

```powershell
uv run codex-multi-launcher
```

也可以临时去掉 `--windowed` 重新打包，这样启动 exe 时会显示控制台错误信息。

### 4. 删除账号失败

通常是对应 Codex 窗口仍在运行，导致目录文件被占用。

处理方式：

1. 关闭该账号对应的 Codex 窗口。
2. 点击“刷新状态”。
3. 再次删除账号。

### 5. 新增账号较慢

新增或导入账号时，启动器会从微软商店版目录复制 Codex 程序目录到该账号的 `CodexPortableApp`，目录较大时会花一些时间。

后续启动会复用该副本；如果微软商店版 Codex 已更新，启动器会在账号未运行时先更新该副本。
