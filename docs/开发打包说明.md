# CodexMultiLauncher 开发与打包说明

本文档说明当前 uv 工程和 Electron 工程并存时的运行、依赖、虚拟环境和打包流程。

## 环境要求

- Windows 10 / Windows 11
- Python 3.11 或更高版本
- uv
- Node.js
- yarn
- Bash（Windows 可使用 Git Bash）
- 已安装的 Codex 桌面端或可执行的 `codex` 命令

检查 uv：

```bash
uv --version
```

检查 Python：

```bash
export UV_PROJECT_ENVIRONMENT="./python/.venv"
uv run python --version
```

检查前端工具：

```bash
node --version

```

## 虚拟环境

本项目使用 uv 管理虚拟环境和依赖。

初始化普通运行环境：

```bash
export UV_PROJECT_ENVIRONMENT="./python/.venv"
uv sync
```

初始化开发和打包环境：

```bash
export UV_PROJECT_ENVIRONMENT="./python/.venv"
uv sync --dev
```

脚本会通过 `UV_PROJECT_ENVIRONMENT` 把虚拟环境固定到 `python\.venv`。该目录不需要提交。

## 依赖说明

启动器运行时只依赖 Python 标准库，不需要第三方运行依赖。

开发依赖放在 `pyproject.toml` 的 `dev` 依赖组中：

```toml
[dependency-groups]
dev = [
    "basedpyright>=1.39.9",
    "cython>=3.2.6",
    "pyinstaller>=6.14.0",
]
```

更新锁文件：

```bash
uv lock
```

安装 Electron 界面依赖：

```bash
yarn install
```

## 运行源码

Electron 开发模式：

```bash
yarn dev
```

新界面的页面不会直接读写配置文件，而是通过 `src\preload\index.ts` 暴露的白名单接口调用 `python\bridge\cli.py`。

也可以直接调试 Python 桥接命令：

```bash
export UV_PROJECT_ENVIRONMENT="./python/.venv"
uv run python -m bridge.cli get_app_state "{}"
```

## 语法检查

```bash
export UV_PROJECT_ENVIRONMENT="./python/.venv"
uv run python -m py_compile ./python/main.py
yarn build
```

如果没有输出，表示语法检查通过。

## 打包 Electron 应用

推荐使用脚本：

```bash
yarn build
```

命令会执行：

1. 调用 `scripts/build_backend.sh`，先用 `python/setup_pyd.py` 将 `bridge`、`core` 模块临时编译为 `pyd`，再生成 Python 桥接后端 `resources\main.exe`。
2. 执行 `yarn build:shell` 生成 Electron 主进程、预加载层和渲染层产物。
3. 执行 `yarn build:installer` 生成 Windows 安装包。

兼容入口 `yarn build:win` 仍会转到同一套完整打包流程。

当前新界面开发模式会用 `uv run python -m bridge.cli` 调用 Python；生产包会优先查找 `resources\main.exe`。

后端打包完成后会自动调用 `python/clear_pyd.py` 清理临时 `pyd` 和 `c` 文件，避免影响源码开发。

## 清理构建产物

推荐使用脚本：

```bash
bash ./scripts/clean.sh
```

脚本会清理：

- `build`
- `dist`
- `out`
- `release`
- `*.spec`
- `__pycache__`

## 运行逻辑概览

启动器主要流程：

1. 读取配置文件 `config.json`。
2. 如果主配置损坏，尝试从最近两份备份恢复。
3. 读取账号列表和每个账号资料状态。
4. 用户新增、改名或删除账号。
5. 新增账号时，把当前 `%USERPROFILE%\.codex` 下的 `auth.json` 和 `config.toml` 复制到账号资料目录。
6. 启动账号时，先确认是否已有 Codex 进程运行；如需切换，则提示用户关闭当前进程。
7. 切换前备份当前活动 `auth.json` 和 `config.toml`。
8. 把目标账号资料写入 `%USERPROFILE%\.codex`。
9. 自动解析 Codex 启动方式，优先使用已识别的桌面程序或微软商店应用标识，兜底使用 `codex app`。
10. 通过进程命令行判断 Codex 是否正在运行。
11. 账号列表直接展示认证状态、运行状态和额度缓存。

## 账号资料目录结构

单个账号目录大致如下：

```text
CodexProfiles
└─ 工作号
   ├─ auth.json
   └─ config.toml
```

## 常见问题

### 1. `uv` 找不到

先安装 uv，然后重新打开终端。

安装方式可以参考 uv 官方文档；如果本机已安装，`uv --version` 应能输出版本号。

### 2. 打包时提示找不到 PyInstaller 或 Cython

执行：

```bash
export UV_PROJECT_ENVIRONMENT="./python/.venv"
uv sync --dev
```

然后重新运行：

```bash
yarn build:backend
```

### 3. 打包后 exe 打不开或闪退

可以先用开发模式确认问题：

```bash
yarn dev
```

也可以临时去掉 `--windowed` 重新打包，这样启动 exe 时会显示控制台错误信息。

### 4. 删除账号失败

通常是对应 Codex 窗口仍在运行，导致目录文件被占用。

处理方式：

1. 关闭该账号对应的 Codex 窗口。
2. 点击“刷新状态”。
3. 再次删除账号。

### 5. 新增账号失败

新增账号会读取当前 `%USERPROFILE%\.codex\auth.json`。如果该文件不存在，说明默认 Codex 尚未完成登录，请先用默认 Codex 登录后再保存账号。

- `uv sync --dev` 成功。

- `yarn dev` 能打开界面。

- 账号状态能显示“运行中”“就绪”“认证已保存”或“认证缺失”。
- 切换账号前能提示关闭正在运行的 Codex。
- 删除账号时，对应目录能被清理。
- `yarn build:backend` 能生成新版 `resources\main.exe`。
