# Codex 多账号切换器使用说明

## 基本模型

本工具只做账号切换。

它使用系统默认安装的 Codex 程序，不复制 Codex 程序目录。每个账号只保存两类资料：

- `auth.json`
- `config.toml`

切换账号时，工具会把目标账号资料写入当前用户的 `.codex` 目录：

```text
%USERPROFILE%\.codex\auth.json
%USERPROFILE%\.codex\config.toml
```

然后执行：

```bash
codex app
```

## 不支持的功能

当前版本不再提供：

- 程序副本复制。
- 多账号并发多开。
- 会话同步。
- 记忆同步。
- 手动设置 Codex 程序路径。
- 程序副本版本更新。

## 首次准备

先确认默认 Codex 命令可用：

```bash
codex --version
```

如果命令不可用，请先安装 Codex，并确保命令已加入系统路径。

## 保存当前账号

1. 先用默认 Codex 完成登录。
2. 打开本工具。
3. 进入“账号”页面。
4. 点击“新增账号”。
5. 输入账号名称。
6. 工具会把当前 `.codex` 下的 `auth.json` 和 `config.toml` 保存到该账号资料目录。

## 切换并启动账号

1. 在“账号”页面选择目标账号。
2. 点击“切换并启动”。
3. 如果检测到 Codex 正在运行，工具会提示确认。
4. 确认后，工具会先关闭当前运行的 Codex。
5. 工具会备份当前活动配置。
6. 工具会写入目标账号资料。
7. 工具会执行 `codex app`。

如果取消确认，工具不会关闭当前 Codex，也不会切换账号。

## 关闭 Codex

账号处于运行状态时，按钮会显示为“关闭”。点击后会关闭检测到的 Codex 相关进程。

## 账号资料目录

默认账号资料根目录：

```text
%USERPROFILE%\Documents\CodexProfiles
```

示例：

```text
CodexProfiles
├─ 工作账号
│  ├─ auth.json
│  └─ config.toml
├─ 备用账号
│  ├─ auth.json
│  └─ config.toml
```

可以在“设置”页面更改账号资料根目录。更改时会迁移整个账号资料根目录。

## 删除账号

“删除”会移除工具中的账号入口，并删除该账号资料目录。

如果该账号是当前账号且 Codex 正在运行，请先关闭 Codex。

## 修改账号名称

“改名”会修改工具中的账号显示名称，并移动对应账号资料目录。

如果 Codex 正在运行，请先关闭后再改名。

## 配置文件

工具配置保存在：

```text
%LOCALAPPDATA%\CodexMultiLauncher\config.json
```

工具会保留最近两份正常配置备份：

```text
%LOCALAPPDATA%\CodexMultiLauncher\config.json.last-good.json
%LOCALAPPDATA%\CodexMultiLauncher\config.json.prev-good.json
```

如果主配置损坏，工具会优先从备份恢复。
