# Codex 多账号启动器说明

## 使用方式

1. 双击 `dist\CodexMultiLauncher.exe`。
2. 如果系统弹出管理员权限确认，请选择允许。
3. 启动器会自动识别微软商店版 Codex 主程序，并在页面显示商店版地址和版本号。
4. 如果要沿用当前 Codex 的非登录配置，请点击“导入原本账号”。
5. 如果要添加新账号，请点击“新增账号”。
6. 点击对应账号的“启动”按钮。
7. 账号行会显示“运行中”“就绪”“需处理”或“异常”。
8. 需要重新检查运行状态时，点击“刷新状态”。
9. 需要查看环境、路径、权限和账号目录状态时，点击“诊断”。
10. 需要调整显示名称时，点击该账号右侧的“改名”。
11. 不再需要某个账号时，点击该账号右侧的“删除”。
12. 新账号首次启动后，在对应 Codex 窗口里登录不同账号。

## 实验性同步会话

如果希望多个账号窗口看到同一批 Codex 会话，可以勾选“实验性同步会话”。

开启后，后续启动的账号会把以下路径挂到同一个共享目录：

- `CodexHome\sessions`
- `CodexHome\attachments`
- `CodexHome\ambient-suggestions`
- `CodexHome\session_index.jsonl`

桌面端侧栏使用的 `CodexHome\state_5.sqlite` 不会做实时共享链接。启动器会在启动账号前把本地侧栏状态合并到共享状态库，再复制回该账号的本地真实 SQLite 文件。

项目列表还依赖 `CodexHome\config.toml` 中的 `[projects.'...']` 段。启动器只会合并这些项目段，不会同步整个 `config.toml`，避免账号专属的 MCP、插件、Token 和桌面偏好互相覆盖。

同步模式下，启动器还会确保 `[desktop] ambient-suggestions-enabled = true`。Codex 桌面端的项目侧栏依赖这个开关；关闭时可能只显示当前项目或少量项目。

项目侧栏还依赖 `CodexHome\cap_sid` 中的 `workspace_by_cwd` 工作区注册表。启动器只合并这一项，不同步账号级 `workspace`、`readonly` 和 `writable_root_by_path` 权限状态。

首次为某个账号启用同步时，启动器会尽量把该账号已有的会话目录、项目建议目录、索引、项目白名单、工作区注册表和桌面端侧栏状态合并到共享目录，并把被替换的本地路径备份为 `.local-before-sync` 后缀。

以下内容不会同步：

- `auth.json`
- `.env`
- `config.toml`
- `plugins`
- `browser`
- `sqlite`
- `logs_*.sqlite`

也就是说，同步会话不会共享登录态、账号完整配置、记忆、插件或浏览器数据。

注意：该功能仍是实验性的。侧栏状态会在启动账号时同步，不是运行中实时同步。遇到异常时，可以先关闭所有 Codex 窗口，再从启动器重新启动账号，让启动器重新检查共享路径。

## 实验性同步记忆

如果希望多个账号合并 Codex 的“记忆”，可以额外勾选“实验性同步记忆”。

开启后，启动器会在启动账号前合并以下数据库：

- `CodexHome\memories_1.sqlite`
- `CodexHome\sqlite\memories_1.sqlite`，如果该路径存在

目前只合并记忆相关表：

- `jobs`
- `stage1_outputs`

记忆数据库不会做实时共享链接。启动器会先把各账号的健康数据库合并到共享目录里的 `memories_1.sqlite`，再复制回当前账号的本地真实 SQLite 文件。

注意：记忆同步也是启动时同步，不是运行中实时同步。建议关闭所有 Codex 窗口后，再用启动器依次重新启动需要同步的账号。

## 管理员权限

当前版本启动时会请求管理员权限。

管理员权限用于提升启动器复制文件、启动进程和访问目录的能力，但不等同于恢复微软商店应用身份。

## 微软商店版说明

启动器会自动从 `C:\Program Files\WindowsApps` 识别微软商店版 Codex，并为每个账号复制一份普通目录中的 Codex 程序副本。

这是为了避免直接执行商店应用时出现“拒绝访问”，并让独立账号目录真正传给 Codex 后台服务。

新增或导入账号时，会在该账号目录下生成独立的 `CodexPortableApp` 程序副本，耗时会稍长。

启动账号前，启动器会重新识别当前微软商店版 Codex。如果商店版已经更新，且该账号没有运行，会先更新该账号的 `CodexPortableApp` 程序副本，再启动。

如果该账号正在运行，启动器不会替换程序副本。请先关闭该账号窗口，再重新启动以同步新版程序。

不同账号不再共用同一个程序副本，减少程序运行状态、缓存和登录信息互相影响的可能。

## 隔离目录

启动器会为每个账号创建独立目录：

```text
C:\Users\Administrator\Documents\CodexProfiles\账号一
C:\Users\Administrator\Documents\CodexProfiles\账号二
```

每个账号会隔离：

- `APPDATA`
- `LOCALAPPDATA`
- `CODEX_HOME`
- `CodexPortableApp`

## 登录信息隔离

当前版本按“登录信息强隔离”处理：

- 新增账号不会复制任何旧登录信息。
- 导入原本账号只复制非登录配置，不复制登录态。
- 每个账号使用独立程序副本、数据目录和配置目录。

## 导入原本账号

“导入原本账号”会复制当前默认 Codex 的非登录配置到独立账号目录。

登录信息不会复制，导入后的账号首次启动需要在对应 Codex 窗口里单独登录。

原本账号只需要导入一次。新版会记录已导入的原本账号；从旧配置升级时，会默认把列表中的第一个已有账号视为原本账号，避免重复导入。

建议导入前先关闭正在运行的 Codex，避免少量配置文件被占用。

## 删除账号

“删除”会移除启动器中的账号入口，并删除该账号对应的独立数据目录。

如果对应 Codex 仍在运行，部分文件可能被占用。此时请先关闭该 Codex 窗口，再手动清理提示中的目录。

## 修改账号名称

“改名”会修改启动器里的账号显示名称，并同步移动该账号对应的独立数据目录。

如果对应 Codex 仍在运行，目录可能被占用。此时请先关闭对应 Codex 窗口后再改名。

## 运行状态

启动器会根据当前 `Codex.exe` 进程命令行中的账号数据目录判断运行状态。

账号状态含义：

- “运行中”：该账号对应的 Codex 进程仍在运行。
- “就绪”：账号目录、程序副本和基础配置未发现明显问题。
- “需处理”：账号可识别，但存在程序副本需同步、目录尚未创建等提醒。
- “异常”：账号目录、程序副本或关键配置缺失，需要先处理。

刚启动后状态可能需要一两秒更新；可以点击“刷新状态”手动刷新。

## 诊断报告

点击“诊断”会打开当前环境报告，内容包括：

- 配置文件和备份文件状态。
- 账号根目录是否可写。
- 微软商店版 Codex 路径和版本。
- 默认 `.env` 是否存在。
- 每个账号的目录、程序副本、`CodexHome` 和异常提醒。

## 启动器配置

启动器配置保存在：

```text
C:\Users\Administrator\AppData\Local\CodexMultiLauncher\config.json
```

启动器会保留最近两份正常配置备份：

```text
C:\Users\Administrator\AppData\Local\CodexMultiLauncher\config.json.last-good.json
C:\Users\Administrator\AppData\Local\CodexMultiLauncher\config.json.prev-good.json
```

如果主配置损坏，启动器会优先从备份恢复；无法恢复时才使用默认配置。

如果没有自动识别到 Codex，请确认已经从微软商店安装 Codex，并以管理员身份运行启动器。
